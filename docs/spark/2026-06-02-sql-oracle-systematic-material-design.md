# SQL / Oracle 26ai 体系資料 作成設計

作成日: 2026-06-02

## 目的

SQLを体系的に理解するための判断材料資料を、Single HTMLとして作成する。単純な機能一覧ではなく、SQL標準の主要概念、Oracle AI Database 26ai前提の差分、DB設計、DB設定、SQLチューニング、オプティマイザ、ヒント句、はまりポイントを並列に残す。

## 採用アプローチ

推奨案として「標準SQLを骨格、Oracleを実運用レイヤとして重ねる」構成を採用する。

- 代替案1: SQL文法辞典型。網羅性は高いが判断基準が弱くなる。
- 代替案2: Oracleチューニング特化型。実務には近いがSQL標準との切り分けが曖昧になる。
- 採用案: 標準SQL / Oracle固有 / チューニング / 設計を章ごとに切り分け、各章で「何を標準として考えるか」「Oracleでは何が変わるか」「何にはまるか」を示す。

## 成果物

- `sql/sql.html`: Single HTMLの本体。外部CSS/JSなし。
- `sql/research/sources.md`: 調査ログ。最低100件を満たすため、Oracle公式ドキュメントTOCからSQL・チューニング・リファレンス・Conceptsの関連ページを抽出し、ISO/IEC 9075の公式ページも併記する。

## 主要章立て

1. 何についての資料か
2. なぜSQLを標準と実装で分けるべきか
3. 全体地図
4. SQL標準の主要機能
5. Oracle前提で変わる部分
6. DB設計と物理設計
7. SQL実行の仕組みとオプティマイザ
8. チューニング実務
9. ヒント句とSQL Plan Management
10. DB設定・初期化パラメータ
11. はまりポイント集
12. 例: 最小例・実務例・失敗例
13. 参照資料

## 正確性方針

- SQL標準はISO/IEC 9075:2023を基準にする。ただし全文標準仕様は有償であるため、公開されているISO概要とOracle公式ドキュメントを中心に確認する。
- Oracleは2026-06-02時点で最新系としてOracle AI Database 26aiを前提にする。OracleドキュメントURLのパスに `/23/` が含まれても、ページタイトルや内容が26aiとして公開されている場合は26ai扱いにする。
- 断定しすぎない。ヒント句、パラメータ、索引設計は「常に有効」ではなく、条件・副作用・代替策を併記する。
