# AGENTS.md — エージェント運用ルール

このリポジトリは「判断材料ドキュメント生成プロジェクト」。
正式なワークフローは `./PROJECT.md` を参照(常に最新の正)。

ここには、どのエージェント(Codex / Claude いずれも)も毎回守る**不変ルール**だけを置く。

## 不変ルール

1. **単純要約をしない。** 目的は要約ではなく判断材料の集積。情報量を削らない。（アイコンや図、表等情報量が同一な別形式への変換はOK）まとめは個別指示があるときだけ。
2. **選択肢を提示する。** 有力候補を1つに断定せず、相反する見解・代替案を並列で残す。断定箇所を見つけたら選択肢を広げる方向で追加調査する。
3. **調査は最低100件・上限なし。** 著者/媒体/立場を分散させ、偏りを記録して補正する。
4. 質問の内容によって対象概念の範囲（調査範囲）が大きく変わることが考えられる。対象概念が大きすぎる場合等は明記すること。概念内の大分類を記載すること。
5. **SubAgent は生情報をファイルに書き出す。** 親へ要約を返させない。返すのは目録(件数・出典ドメイン・著者・カバー/未カバー論点)のみ。書き出し先は `./<テーマ>/research/<観点グループ>.md`。
6. **調査並列は 4〜6 体。** 近い観点をグループにまとめて割り当てる。
7. **出力は Single HTML。** 全観点を1ファイルに集約し、画像は Base64 埋め込み。目次はあっていいが、アンカーは不要。

## 情報収集

ユーザの指示は雑な方針であるので、**完璧に守る必要はない**。
指示にはないが、考慮すべき情報や、広げるべき内容は推論しなさい。
ユーザの指示が間違っていそうな場合は、指示の方向性や求めるものを推論して作業を実施しなさい。

資料の判断材料の広げ方や情報のまとめ方に関しても参考資料を利用して把握に努めなさい。
